# REST-сервис скоринга времени доставки (Вариант 3)

## 1. Постановка задачи и описание данных
Данный сервис решает задачу **бинарной классификации**: прогнозирование своевременности доставки заказа (`prepared_on_time`) на основе признаков заказа. 

### Целевая переменная:
- `prepared_on_time`: `1` — заказ доставлен вовремя, `0` — заказ задержан.

### Список входных признаков:
- `store_id` (категориальный): Идентификатор ресторана (от 1 до 100).
- `items_count` (числовой): Количество позиций в чеке.
- `order_price` (числовой): Стоимость заказа в рублях.
- `delivery_distance` (числовой): Расстояние от ресторана до клиента (в км).
- `planned_prep_time` (числовой): Запланированное время на сборку и приготовление заказа (в минутах).

---

## 2. Архитектура решения и модель
- **Данные**: Сгенерирован синтетический сбалансированный датасет (5000 строк) со встроенной зависимостью целевой переменной от факторов расстояния, объема заказа и времени подготовки.
- **Предобработка**: 
  - Числовые признаки стандартизированы с помощью `StandardScaler`.
  - Категориальные закодированы методом `OneHotEncoder` (с обработкой неизвестных категорий на инференсе).
- **Модель**: Случайный лес (`RandomForestClassifier`), завернутый в `sklearn.Pipeline`.
- **Обучение**: Произведен автоматический подбор гиперпараметров методом кросс-валидации `GridSearchCV`. Лучшая модель сохранена в бинарный файл `model/delivery_model.joblib`.
